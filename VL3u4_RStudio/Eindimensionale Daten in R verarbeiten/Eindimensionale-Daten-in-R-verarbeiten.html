Package installieren

Vorgang des Installierens eines Packages

In R gibt es sehr viele sogenannte Hilfspakete (“Packages”), die beispielsweise spezielle Funktionen oder auch Beispieldatensätze enthalten. Um ein solches Hilfspaket zu verwenden, muss es zunächst installiert werden:

install.packages("NamedesPackages")

Dieser Vorgang des Installierens ist immer nur EINMALIG notwendig.

Im folgenden Schritt sollen Sie die folgenden beiden Packages installieren:

  • mosaic

  • reshape2

Hinweis: Das Package mosaic enthält einige brauchbare Funktionen, die wir verwenden werden, das Package reshape2 unter anderem einen Beispieldatensatz, den wir benutzen.

Package Laden

Vorgang des Ladens eines Packages

Bei jedem Öffnen von R müssen die installierten Packages geladen werden. Man kann sich das Installieren wie das Anschließen einer Lampe an das Stromnetz vorstellen und das Laden wie das Anschalten der Lampe, wenn man Licht möchte.

Um ein Hilfspaket zu laden schreiben wir:

library(NamedesPackages)

ACHTUNG: Das Laden des Pakets ist jedes Mal aufs neue durchzuführen, wenn dieses gebraucht wird (“Licht anschalten”).

Hinweis: Nicht wundern, in der Ausgabe werden durch das Package-Laden diverse Infos angezeigt (z.B., dass im Hintergrund noch weitere Packages geladen werden), diese brauchen uns erstmal nicht weiter zu interessieren.

Im folgenden Schritt sollen Sie die folgenden Packages laden:

  • mosaic (Paket mit vielen hilfsreichen Befehlen bei der Datenauswertung)

  • reshape2 (Paket mit einem Beispieldatensatz “tips”)

Im folgenden Schritt sollen Sie sich den Datensatz “tips” aus dem Paket “reshape2” anschauen.

Zu Beachten: Das Laden des Pakets muss (wie in der Aufgabe zuvor gelernt) jedes Mal aufs neue durchgeführt werden.

Kennzahlen

Einige Befehle

Um sich einen Überblick über vorliegende Daten zu verschaffen und erste wichtige statistische Kennzahlen (wie zum Beispiel: Minimum, Maximum, Median, Mittelwert, Standardabweichung, versch. Quantile) zu erhalten, können Sie folgende Funktionen nutzen:

  • summary Bsp.: summary(tips)

  • favstats Bsp.: favstats(~tip, data = tips)

  • inspect Bsp.: inspect(tips)

Die beiden Funktionen summary und inspect sind hierbei jeweils Funktionen des Packages “mosaic”.

Verschaffen Sie sich nun einen Überblick über den Datensatz mittels der oben genannten Funktionen.

library(reshape2)

Modus bestimmen

Beim Modalwert oder auch Modus genannt handelt es sich in der Statistik um jene Ausprägung einer Variable, die am häufigsten vorkommt. Der Modalwert ist also der häufigste Wert einer Verteilung. Er ist somit jener Wert, der „typisch“ für eine Verteilung ist und entspricht dem höchsten Punkt der Verteilung.

Allerdings: Es gibt keinen einfachen Befehl für den Modus, dieser ist aber leicht aus einer Häufigkeitstabelle ablesbar, z.B. mit tally aus mosaic !

Aufgabe: Bestimmen Sie an welchem Wochentag am häufigsten Trinkgeld gegeben wurde.

library(mosaic)
library(reshape2)
#tally(~NamedesMerkmals, data = NamedesDatensatzes)

Boxplot

Der Box-Plot (auch Box-Whisker-Plot oder deutsch Kastengrafik) ist ein Diagramm, das zur grafischen Darstellung der Verteilung eines mindestens ordinalskalierten Merkmals verwendet wird. Es fasst dabei verschiedene robuste Streuungs- und Lagemaße in einer Darstellung zusammen.

Im folgenden Beispiel zeigen wir euch wie ihr in Abhängigkeit zweier Merkmale ein Boxplotfenster erstellen könnt, um anschaulich Zusammenhänge und mögliche Korrelationen zu verdeutlichen.

Führen Sie hierzu lediglich den schon im Code-Feld enthaltenen Code aus und interpretieren Sie für sich das Ergebnis, welches Sie nach kurzem Runterscrollen sehen können.

library(mosaic)
library(reshape2)
boxplot(total_bill~time, data = tips)
